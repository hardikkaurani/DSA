## `// about`

Not just solving problems — **engineering a thinking system**.

Every problem is a signal. Every mistake is a data point.  
This repository is compiled daily, like good software.

---

## `// architecture`
```
DSA/
├── arrays/           → foundation of everything
├── hashing/          → O(1) access, fast lookups
├── sliding-window/   → range optimization
├── stack/            → structured LIFO flow
└── recursion/        → thinking in subproblems
```

---

## `// approach`

| # | Layer | What it means |
|---|-------|---------------|
| 01 | **Pattern Recognition** | Every problem maps to a known structure. Find it before writing a line. |
| 02 | **Feedback Loop** | Wrong approaches aren't failures — they're the loop tightening. |
| 03 | **Daily Compounding** | One problem, every single day. Cognitive returns compound. |
| 04 | **Depth over Speed** | Understand *why* the solution works. Not just that it does. |

---

## `// system`
```
01 → Identify the pattern         (array? graph? DP?)
02 → Brute force first            (understand before optimizing)
03 → Optimize step by step        (what's the bottleneck?)
04 → Write clean code             (code is read more than written)
05 → Repeat relentlessly          (until the pattern is instinct)
```

---

## `// philosophy`

> Most engineers solve problems to **finish them**.

> The difference compounds over time.
